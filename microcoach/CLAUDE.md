# MicroCoach — Claude Context

## LaTeX Support Task

### Overview
The goal is to replace Unicode math output from the LLMs with properly typeset LaTeX, and render it on the frontend using KaTeX.

Previously all prompts instructed the LLMs to output plain Unicode (e.g. x², √x, ÷). The task flips this to LaTeX with `$...$` inline and `$$...$$` display delimiters, then renders them in React.

### Frontend Changes

#### `prototype/src/components/MathText.js` (new file)
A shared React component that renders strings containing LaTeX math expressions.
- Splits strings on `$$...$$` (display/block math) and `$...$` (inline math) delimiters
- Renders math segments via KaTeX (`katex.renderToString`)
- Renders plain text segments as-is
- Falls back gracefully to raw text if KaTeX cannot parse an expression
- KaTeX installed as a dependency in `prototype/package.json`

#### `prototype/src/components/NextStepDetailsModal.js`
Updated all math-bearing render sites to use `<MathText>` instead of plain string rendering:
- `move.targets`
- `move.instructionalMove`
- `whatStudentsDo` / `whatYouDo` bullet details
- `activitySteps.problem`
- `incorrectWorkedExamples[].problem`
- `WorkedExampleText` — each line (including annotation lines) now goes through `MathText`
- `coreActivity` items
- `discussionQuestions` items

### Dev Workflow Notes

#### Running the seed scripts (e.g. `yarn generate`)
- Must be run from `microcoach/api`
- Uses `ts-node` directly (not `npx ts-node`) — workspace hoisting makes npx resolve to the wrong path
- AppSync config is auto-loaded from `src/aws-exports.js` (generated by Amplify) — no env vars needed
- AWS IAM credentials must be present in `~/.aws/credentials`. Use `muhammad-profile` if the default profile fails:
  ```powershell
  $env:AWS_PROFILE="muhammad-profile"
  yarn generate
  ```

#### `api/src/seed/appsync-config.ts`
Modified to support three credential sources in priority order:
1. `APPSYNC_ENDPOINT` + `APPSYNC_API_KEY` env vars
2. `src/aws-exports.js` (auto-loaded, no env vars needed)
3. AWS Secrets Manager via `APPSYNC_SECRET_NAME` env var (original behavior)
